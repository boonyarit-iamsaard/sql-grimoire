# 05 — Playtest protocol

Status: ready-for-human
Blocked by: 04

## Problem

The app has no telemetry, no feedback channel, and no analytics — by design (`VISION.md`
out-of-scope list). All evidence therefore comes from live observation, and the protocol must
exist in writing **before** the first session so the verdict cannot be renegotiated
afterward.

## Contents to write (founder)

1. **Recruiting note**: two or three working developers who use SQL occasionally but are not
   database specialists (the `PRODUCT.md` target user). Cold players only — no one who has
   seen the project.
2. **Session script**: what is said at the start (almost nothing beyond "think aloud"), what
   is never done (helping, explaining, defending), and the closing questions.
3. **Observation checklist** (from `README.md`, "Evaluating the prototype"): dialogue read or
   skipped; failure messages helping without revealing; hint usage; and the primary metric —
   behavior at the Arc's end, in front of the locked `???` Location.
4. **Pass/fail criteria, written in advance.** Draft for the founder to edit: at least two of
   three testers finish the Arc unassisted (hints allowed, intervention not) AND ask about
   the next Mission unprompted. Anything less is a format finding, not a green light for the
   constraints-and-transactions arc.

## Acceptance

The protocol document exists in `.scratch/guild-ledger-arc/` before the first external
session is scheduled; every session's observations are appended to it afterward.
